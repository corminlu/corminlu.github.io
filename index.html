2016-07-23

使UIImageView正确显示图片

经常会遇到使用的图片和UIImageView的frame比例不一样的时候，出现图片变形、白边等情况，这个时候可以通过UIImageView的contentMode属性来解决。

UIViewContentMode

UIViewContentMode主要可用以下几个属性来改变图片：

  • UIViewContentModeScaleToFill 缩放图片来填充UIImageView的frame，图片高宽比例和UIImageView的frame不一致的时候，会调整图片比例，然后会出现图片拉伸变形
  • UIViewContentModeScaleAspectFit 保持图片比例，按图片最大的边去缩放图片大小，图片高宽比例和UIImageView的frame不一致的时候，可能会出现高度或宽度不够的无法完全填充UIImageView，然后出现白边
  • UIViewContentModeScaleAspectFill 保持图片比例，按图片最小的边去缩放图片大小，图片高宽比例和UIImageView的frame不一致的时候，虽能完全填充UIImageView，但图片有一部分不会被显示出来

2016-06-25

.Net Core Segmentation Fault错误

最近升级到.Net Core RC2，在Windows、Mac以及Linux Docker容器里运行正常，一发布到Linode上面的CentOS，运行没一会儿就会出现Segmentation Fault错误。

项目里用到了Quartz.Net，而Quartz.Net也有适配.Net Core的分支quartznet-3，虽然可能有不少bug，但目前已经能正常使用。

试着把Quartz.Net相关功能去掉，在Linode上的CentOS运行，过了一会儿还是因为Segmentation Fault终止了。

直到有一天登录Linode管理控制台，无意中发现当前Configuration Profile的”VM Mode”是Paravirtualization（即半虚拟化），而我在很早之前就把vps从Xen升级到了KVM，但一直没注意到VM Mode设置，通过一番搜索后决定直接把VM Mode改成Full-Virtualization（全虚拟化）。

抱着试一试的态度重新运行了一下.Net Core程序，居然没有出现Segmentation Fault，过了几个小时然后又过了几天，一直在正常运行。

虽然不知道是什么原因（很可能就是虚拟化的问题），总算可以在Linode上运行.Net Core应用了。

2016-04-17

Visual Studio内置Node版本过低导致Bug

今天把gulp-inject升级到4.0，结果用vs运行gulpfile任务时出现bug

gulp-inject\src\inject\index.js:127

折腾了半天还以为是api变更导致的问题，最后找到StackOverflow上面有人遇到同样的问题，原来是在vs运行会用Common7\IDE\Extensions\Microsoft\Web Tools\External\node目录下的老版本node(0.10.xx)来执行gulp任务，结果导致问题，用他提供的办法解决了，把最新版5.1xx的路径加入到External Web Tools中即可。

下次遇到需要在vs里执行需要node环境的任务时，一定要注意node版本问题，如npm/bower包管理、gulp任务等。
