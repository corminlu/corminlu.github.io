迁移到Asp.Net Core RC2

  • aspnet
  • dev
  • dotnet core

本文可能仅适用于.Net Core RC2，据说RTM会把目前的一些json配置文件给废弃掉，以及csproj回归

.Net Core RC2发布将近一个月，从DNX到RC2，API经过比较大的变化，不过目前基本上都稳定下来了，试着将DNX WebApi项目迁移了一下，运行还比较稳定。估计等到RTM发布不会有太大都变动，现在可以开始尝尝鲜了。

运行环境

DNVM/DNX/DNU都已经被废弃了，现在有了全新的工具DotNet CLI，以及统一的域名dot.net，找到对应平台的运行时安装即可。

  • Windows （选择一个即可）
    • 运行时+CLI工具：.NET Core SDK Installer
    • SDK+VS集成：.NET Core Tooling Preview 1 for Visual Studio 2015

  • CentOS 运行以下命令

    • 需要先安装libunwind，全新安装以来组件见文档

    • 接着安装sdk

  • OSX

    • 可选，删除老版本cli

    • 需要先安装openssl

    • 接着下载安装.Net Core SDK Installer

如果要尝试最新版，直接在DotNet CLI下载对应版本即可。

DNVM未来说不准还会回归，用来管理运行时，现在这样下载安装确实很不方便

第三方库

目前很多第三方库不兼容.Net Core，要么等它们升级，要么只能自己拿源码来改造了。

比如Quartz.net，有支持.Net Core的分支正在开发中，可以直接拿来用。

比如Jumony，没有依赖什么底层库，很容易改造成支持.Net Core。

而EPPlus就不行了，依赖一大堆.Net Core不支持的库，只能保留最基本的读取文件功能了。

另外还有些库根本没办法改造，只能找替代产品了。
