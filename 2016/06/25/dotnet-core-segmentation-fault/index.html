.Net Core Segmentation Fault错误

  • aspnet
  • centos
  • dev
  • dotnet core
  • linode

最近升级到.Net Core RC2，在Windows、Mac以及Linux Docker容器里运行正常，一发布到Linode上面的CentOS，运行没一会儿就会出现Segmentation Fault错误。

项目里用到了Quartz.Net，而Quartz.Net也有适配.Net Core的分支quartznet-3，虽然可能有不少bug，但目前已经能正常使用。

试着把Quartz.Net相关功能去掉，在Linode上的CentOS运行，过了一会儿还是因为Segmentation Fault终止了。

直到有一天登录Linode管理控制台，无意中发现当前Configuration Profile的”VM Mode”是Paravirtualization（即半虚拟化），而我在很早之前就把vps从Xen升级到了KVM，但一直没注意到VM Mode设置，通过一番搜索后决定直接把VM Mode改成Full-Virtualization（全虚拟化）。

抱着试一试的态度重新运行了一下.Net Core程序，居然没有出现Segmentation Fault，过了几个小时然后又过了几天，一直在正常运行。

虽然不知道是什么原因（很可能就是虚拟化的问题），总算可以在Linode上运行.Net Core应用了。
