AppStore上架准备

  • dev
  • ios

记录一下上架到AppStore的准备工作

Developer Center证书相关

创建CSR

后面会用到，过程略

创建AppId

设置好需要使用的功能，比如我需要Push Notification、Associated Domains等

创建证书

分别需要创建Developement和Distribution类型证书

  • iOS App Development 日常开发使用的
  • Apple Push Notification service SSL (Sandbox) 开发时使用的APNS推送证书
  • App Store and Ad Hoc 上架和adhoc内测使用的正式证书
  • Apple Push Notification service SSL (Sandbox & Production) 正式使用以及adhoc内测使用的推送证书

创建证书时为每个证书取好名字，如-appstore、-dev后缀以便区分，然后下载到本地导入到计算机即可。如果要给别人使用，需要导出成.p12格式。

创建Provisioning Profile

根据需要创建

  • iOS App Development 日常开发使用
  • Ad Hoc adhoc内测使用
  • App Store 正式上架使用

创建时同样取好名字以便区分，然后下载双击导入即可

Build项目生成相关

在XCode登录当前的Apple Id，并设置XCode的Team。

Code Sign

找到Build Setting选项卡

  • 在Release的CODE_SIGN_IDENTITY为当前企业名称(“iPhone Distribution: #Company Name# Co., Ltd. (#TEAMID#)”;
  • 设置PROVISIONING_PROFILE为App Store用的Provisioning Profile
  • 确保CODE_SIGN_RESOURCE_RULES_PATH的内容为空，如果不为空，如"$(SDKROOT)/ResourceRules.plist";，则移除内容

Bitcode

如果项目没有使用不支持BitCode的第三方库，确保

移除未使用的库

未使用的库移除掉，然后检查以下设置

如果路径中的库已经不存在，则移除无效的项

BuildVersion

使用Git提交次数来作为版本号，在BuildPhase添加ShellScript

Remove Unused Target

打包上传的时候可能会提示”unsupported architecture ‘[x86_64, i386]’”等，需要在BuildPhase添加ShellScript

BreakPoints

移除所有断点

remove unused resource

移除未使用的资源如图片等，减小包的大小

Universal Link

ios9开始支持applinks设置，在项目的Capabilities开启Associated Domains选项，然后在domains填入

如果有多个域名就填入多个，然后会自动生成.entitlements文件。

另外还需要在AppDelegate处理UrlScheme

在服务器的.well-known目录下放入apple-app-site-association文件，文件内容

appID替换成自己的如ABCD1234.com.apple.wwdc，并确保服务端输出的MIME为application/json。

Https

启用Https保证安全，详见之前的博客。另外确保NSAllowsArbitraryLoads开关是关闭的。

webP/APNG

WebP格式/APNG格式能有效的减小图片资源的大小，有条件的情况下能使用webP就尽量使用，YYImage对WebP/APNG都有很好的支持，可以大胆放心的使用这个库来呈现本地、远程图片。
