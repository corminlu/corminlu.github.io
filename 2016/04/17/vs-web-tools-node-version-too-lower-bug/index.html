Visual Studio内置Node版本过低导致Bug

  • dev
  • vs

今天把gulp-inject升级到4.0，结果用vs运行gulpfile任务时出现bug

gulp-inject\src\inject\index.js:127

折腾了半天还以为是api变更导致的问题，最后找到StackOverflow上面有人遇到同样的问题，原来是在vs运行会用Common7\IDE\Extensions\Microsoft\Web Tools\External\node目录下的老版本node(0.10.xx)来执行gulp任务，结果导致问题，用他提供的办法解决了，把最新版5.1xx的路径加入到External Web Tools中即可。

下次遇到需要在vs里执行需要node环境的任务时，一定要注意node版本问题，如npm/bower包管理、gulp任务等。
